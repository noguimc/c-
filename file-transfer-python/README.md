# 文件互传系统 (Python版)

一个基于Flask的文件互传程序，具有edx-ui风格的界面，支持文件上传、下载和删除功能，最大支持1GB文件。

## 功能特点

- 🎨 采用edx-ui风格设计，界面美观现代
- 📁 支持多文件上传
- 🗂️ 文件列表展示与管理
- 🔍 文件搜索功能
- 📊 实时上传进度显示
- 📱 响应式设计，支持移动设备
- 🗑️ 文件删除功能
- 📥 文件下载功能
- 🚀 可打包为独立exe文件
- 💾 支持最大1GB文件上传

## 技术栈

- 后端：Python, Flask
- 前端：HTML5, CSS3, JavaScript (ES6+)
- 打包工具：PyInstaller

## 安装与运行

### 前提条件

- Python 3.7 或更高版本

### 运行方式一：直接运行Python脚本

1. 克隆或下载项目文件到本地
2. 在项目目录中打开终端
3. 安装依赖：

```bash
pip install -r requirements.txt
```

4. 运行应用（使用默认端口5000）：

```bash
python app.py
```

或者指定自定义端口：

```bash
python app.py --port 8080
```

或使用短参数：

```bash
python app.py -p 8080
```

如果不想自动打开浏览器，可以添加--no-browser参数：

```bash
python app.py --no-browser
```

5. 打开浏览器，访问显示的地址（通常是 http://127.0.0.1:5000）

### 运行方式二：打包为exe文件

1. 克隆或下载项目文件到本地
2. 在项目目录中打开终端
3. 运行打包脚本：

```bash
python build_exe.py
```

4. 打包完成后，在`dist`目录下会生成`文件互传系统.exe`
5. 双击exe文件即可运行程序（使用默认端口5000）

或者通过命令行运行并指定端口：

```cmd
文件互传系统.exe --port 8080
```

或使用短参数：

```cmd
文件互传系统.exe -p 8080
```

如果不想自动打开浏览器，可以添加--no-browser参数：

```cmd
文件互传系统.exe --no-browser
```

## 使用说明

### 上传文件

- 点击"选择文件"按钮选择文件上传
- 或直接将文件拖放到上传区域
- 支持同时上传多个文件
- 上传过程中会显示实时进度
- 支持最大1GB的文件

### 下载文件

- 在文件列表中找到要下载的文件
- 点击文件右侧的下载图标
- 文件将自动下载到默认下载位置

### 删除文件

- 在文件列表中找到要删除的文件
- 点击文件右侧的删除图标
- 确认删除操作

### 搜索文件

- 在搜索框中输入文件名
- 文件列表将实时过滤显示匹配的文件

## 目录结构

```
├── build_exe.py        # 打包脚本

│   ├── css/
│   │   └── style.css  # 样式文件
│   └── js/
│       └── script.js  # 前端脚本
├── uploads/           # 上传文件存储目录（运行时创建）

```

## 注意事项

- 上传的文件存储在服务器本地的`uploads`目录中
- 如果上传同名文件，系统会自动添加时间戳避免覆盖
- exe文件首次运行时，会在exe所在目录创建uploads文件夹用于存储上传的文件
- 最大支持上传1GB的文件
- 默认开放5000端口，可在app.py中修改port变量

## 故障排除

1. 如果运行exe文件时出现错误，请确保：
   - exe文件所在的目录有写入权限
   - 防火墙没有阻止程序访问网络

2. 如果无法访问网页，请检查：
   - 程序是否成功启动（查看控制台输出）
   - 端口是否被其他程序占用

3. 如果文件上传失败，请检查：
   - 文件大小是否超过1GB限制
   - 磁盘空间是否足够
